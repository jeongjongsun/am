# 화주(법인) 정보

## 개요

- **경로**: `/home/basic/shipper-corp`
- **역할**: `om_corporation_m` 기반 법인(화주) 목록 조회·그리드 셀 편집 저장(등급 ADMIN).
- **API**: `GET /api/v1/admin/corporations` (페이징), `PATCH /api/v1/admin/corporations/{corporationCd}` (본문 검증 후 갱신, 성공 시 `om_log_m` 감사 기록) — 응답 래퍼는 `docs/guide/02-개발-표준.md`.
- **UI**: `DataGrid` / `DataGridPaginationFooter`, `PageLayout`, Excel 받기, 행 선택 체크박스 없음. 편집 가능: 법인명·사업자등록번호·대표자명·주소·전화·팩스·대표이메일·홈페이지·비고·사용여부(한 클릭 편집, 포커스 이탈 시 저장 요청). 하단 페이지 크기 옵션 `100, 1000, 5000, 10000`.
- **부가정보**: `corporation_info` JSONB는 서버에서 주요 키만 펼쳐 컬럼으로 표시한다 (`docs/ddl/om_corporation_m.sql` 주석 참고).

## 프로세스 흐름

1. 사용자가 메뉴 **기초정보 관리 → 화주(법인) 정보**로 탭을 연다.
2. 프론트는 `GET /api/auth/me`로 세션을 확인하고, 등급이 `ADMIN`일 때만 목록 API를 호출한다.
3. `GET /api/v1/admin/corporations`로 페이징된 행을 받아 그리드에 바인딩한다.
4. 사용자는 셀을 수정한 뒤 포커스를 벗어나면 `PATCH`로 저장되며, 실패 시 서버 메시지를 보여 주고 목록을 다시 불러온다.
5. Excel 받기·페이지 크기·이전/다음 페이지로 목록을 탐색한다.

## API

| 메서드·경로 | 용도 |
|-------------|------|
| `GET /api/v1/admin/corporations` | 법인 목록 페이징 (`sort` 기본 `corporationCd,ASC`; 허용: `corporationCd`, `corporationNm`, `createdAt`) |
| `PATCH /api/v1/admin/corporations/{corporationCd}` | 법인명·`corporation_info` 표준 키·`is_active` 갱신 (404: `ERR_CORPORATION_NOT_FOUND`) |

## DB

- `om_corporation_m`: `corporation_cd`, `corporation_nm`, `corporation_info`, `is_active`, 감사 일시·작성자 컬럼.

## QA 체크리스트

- [ ] 비관리자(ADMIN 아님)는 화면에서 안내 문구만 보이고 API는 403인지 확인한다.
- [ ] 관리자는 그리드에 `corporation_info`에서 펼친 컬럼이 비어 있을 때도 오류 없이 조회되는지 확인한다.
- [ ] 페이지 크기 변경 시 첫 페이지로 리셋되는지, 마지막 페이지 버튼이 총 건수와 맞는지 확인한다.
- [ ] Excel 받기로보낸 파일이 화면 컬럼과 대략 일치하는지 확인한다.
- [ ] 그리드에서 값을 바꾼 뒤 저장이 되고 `om_log_m`에 `CORPORATION_UPDATE`가 쌓이는지 확인한다.
