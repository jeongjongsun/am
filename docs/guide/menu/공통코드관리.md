# 공통코드 관리

## 개요

`OM_CODE_M`에 등록된 **코드 그룹(대분류)**과 **그룹별 코드값(하위)**을 조회·등록·수정하는 화면입니다. 모든 업무 화면의 드롭다운·필터 등에서 참조하는 공통코드를 여기서 유지합니다.

- **삭제**: 행을 **물리 삭제하지 않습니다.** 사용 중지는 `code_info.use_yn = N` 으로 처리합니다.

## 경로·구성

| 경로 | 설명 |
|------|------|
| `/home/service/common-code` | 공통코드 관리 본화면 (단일 그리드 + 모달) |

> 이전 경로 `/home/admin/common-code` 는 동일 화면으로 리다이렉트됩니다.

## 화면 구성

1. **상단 영역**
   - **페이지 제목**(`h2`)만 둔다. (브레드크럼·요약 리드 문단은 두지 않는다.)
   - **검색**: 카드·패널 없이 페이지 배경 위에 배치. Phoenix `form-icon-container` + `form-floating` + 왼쪽 검색 아이콘. **공통코드**(그룹 ID·하위 `sub_cd`) 및 **코드명**(`code_nm` 다국어 값) 부분 일치.
   - **대분류 등록** 버튼: 모달로 새 그룹 등록.

2. **단일 그리드 ([AG Grid](https://www.ag-grid.com/react-data-grid/getting-started/) Community)**  
   - **대분류** 행(`main_cd = 'CODE'` 의 `sub_cd` = 그룹 ID)과 **하위 코드** 행(해당 `main_cd` 아래 `sub_cd`)을 **동일한 표**에 표시한다.  
   - 초기에는 **대분류 행만** 표시한다.  
   - 하단 **페이징**은 사용자 정보 화면과 동일하게 `DataGridPaginationFooter`를 사용하며, 페이지 크기 옵션은 `100, 1000, 5000, 10000`이다(클라이언트 슬라이스).

3. **열 구성 (요약)**  
   - **관리**: 대분류 행에만 **`+` / `−` 버튼**(펼침·접기)과 **하위 등록** 버튼.  
   - **구분**: 대분류/하위를 Phoenix `badge` 로 표시. **사용·순서** 열은 헤더·값 가운데 정렬.  
   - **공통코드**: 값은 **`btn btn-link me-1`** (`mb-1` 없음). 본문은 `<span>` 만 두어 그룹·표시명 열과 동일한 산세리프·크기로 보이게 한다. 밑줄·점선 등 별도 장식은 두지 않는다. 클릭 시 해당 행 **수정 모달**(대분류·하위 각각)을 연다. (별도 「수정」열은 없음.)  
   - **표시명**: 그리드 **인라인 편집**. 편집 대상은 **현재 UI 언어**에 대응하는 `code_nm` 키(`ko` / `en` / `ja` / `vi`) 한 곳뿐이다. 네 언어 모두 비면 저장을 거부한다.  
   - **사용**·**순서**: 그리드 인라인 편집(`Y`/`N` 선택, 0 이상 정수). 이 두 열에는 **열 필터(헤더 필터)**를 두지 않는다.  
   - 표시 순서 일괄 재배열 API(`PUT .../items/order`)는 클라이언트에 함수만 있고, 본 화면 UI에서는 호출하지 않는다. 순서는 인라인 `disp_seq` 또는 모달에서 조정한다.

4. **검색 시 동작**  
   - 검색어가 있을 때는 모든 그룹에 대해 하위 목록을 조회한 뒤, 매칭되는 행만 표시한다(그룹·하위 혼합).

5. **등록·수정**  
   - **대분류 등록** / **하위 등록**: 모달.  
   - **그룹·하위 수정**: **공통코드 링크**로 모달을 열어 다국어 전체·코드 ID 등을 편집하거나, **표시명·사용·순서**를 그리드에서 직접 편집한다. 인라인 편집은 셀 편집 종료(포커스 이탈 등) 시 `PUT /api/v1/admin/common-codes/{mainCd}/{subCd}` 로 저장된다.

## API

| 메서드·경로 | 용도 |
|-------------|------|
| `GET /api/v1/admin/common-codes/groups` | 코드 그룹 목록 (`CODE` 하위) |
| `GET /api/v1/admin/common-codes/{mainCd}/items` | 해당 그룹의 코드값 목록 (`mainCd`는 `CODE` 불가) |
| `POST /api/v1/admin/common-codes/groups` | 그룹 등록 (`main_cd=CODE`, `sub_cd=groupId`) |
| `POST /api/v1/admin/common-codes/{mainCd}/items` | 코드값 등록 |
| `PUT /api/v1/admin/common-codes/{mainCd}/{subCd}` | 코드명·`use_yn`·`disp_seq` 수정 (그룹 메타·코드값 공통) |
| `PUT /api/v1/admin/common-codes/{mainCd}/items/order` | 본문 `{ "orderedSubCds": ["A","B",...] }` 로 표시 순서 일괄 저장 (현재 본 화면에서 미사용) |

응답은 프로젝트 공통 `ApiResponse` (`success`, `data`, `message`, `code`). 실패 시 `code` 예: `ERR_COMMON_CODE_DUPLICATE`, `ERR_COMMON_CODE_GROUP_UNKNOWN` 등.

## DB·데이터

- 테이블: `om_code_m` ([om_code_m.sql](../../ddl/om_code_m.sql))
- 복합 PK: `(main_cd, sub_cd)`
- `code_nm`: JSONB 다국어 (`ko`, `en`, …)
- `code_info`: JSONB — `use_yn` (`Y`/`N`), `disp_seq` (정렬)

## 권한·보안

- REST URL은 `/api/v1/admin/common-codes/**` 이며, 백엔드는 `@PreAuthorize("hasRole('ADMIN')")` 등으로 보호한다. 비인가·만료 시에는 공통 인증 정책(401/403)을 따른다.

## 프로세스 흐름

1. 로그인 후 사이드 메뉴 **서비스 운영 정보 → 공통코드**로 진입한다.
2. 그리드에 대분류만 표시된다. `+`로 하위를 펼치고 `−`로 접는다.
3. **대분류 등록** 또는 **하위 등록**은 모달에서 입력 후 저장한다.
4. **공통코드** 링크를 누르면 해당 행의 수정 모달이 열린다. 저장 시 React Query 캐시를 무효화해 목록이 갱신된다.
5. **표시명·사용·순서**는 셀을 편집(AG Grid 기본: 더블클릭 등)한 뒤 편집을 마치면 같은 `PUT` API로 저장되고, 성공·실패 여부와 관계없이 관련 쿼리를 무효화해 서버와 동기화한다.
6. 검색어가 있으면 전 그룹 하위를 조회·필터링해 표시한다. 하단 페이징으로 페이지 크기·이동을 조정한다.

## QA 체크리스트

- [ ] `ADMIN` 이 아닌 계정은 화면에 금지 안내만 보이고, API 직접 호출 시 403(또는 정책에 맞는 코드)인지 확인한다.
- [ ] 초기 로드 시 대분류만 보이고, `+`로 하위가 펼쳐지며 `−`로 접히는지 확인한다.
- [ ] 검색어 입력 시 로딩 안내가 나오고, 매칭되는 그룹·하위만 표시되는지 확인한다.
- [ ] **공통코드** 링크로 대분류·하위 각각 수정 모달이 열리고, 저장 후 그리드 값이 갱신되는지 확인한다.
- [ ] **표시명** 인라인: 현재 UI 언어에 해당하는 값만 바뀌는지, 모든 언어를 비운 상태로는 저장되지 않는지 확인한다.
- [ ] **사용** 인라인: `Y`/`N` 만 선택 가능하고 저장 후 반영되는지 확인한다.
- [ ] **순서** 인라인: 0 이상 정수만 허용되고, 잘못된 값은 이전 값으로 되돌아가거나 저장되지 않는지 확인한다.
- [ ] **사용·순서** 열 헤더에 AG Grid **열 필터**(돋보기 등)가 나타나지 않는지 확인한다.
- [ ] 인라인 저장 실패 시 에러 메시지가 표시되고, 재조회 후 서버 값과 그리드가 일치하는지 확인한다.
- [ ] 페이지 크기 변경 시 첫 페이지로 리셋되는지, 이전·다음·마지막 버튼이 총 건수와 맞는지 확인한다.
