# 비밀번호 초기화 (관리자)

## 개요

비밀번호 분실 시 **사용자 스스로 이메일로 재설정하는 흐름은 사용하지 않고**, 등급 **`ADMIN`** 인 관리자만 백오피스에서 대상 사용자에게 **임시 비밀번호**를 발급한다.

## 경로·구성

| 경로 | 설명 |
|------|------|
| `/home/admin/password-reset` | 초기화 화면 (로그인·`grade_cd=ADMIN` 필요) |
| 상단 네비 | `ADMIN` 일 때만 「비밀번호 초기화」 링크 표시 |

## 화면 기능

- 초기화할 **사용자 아이디(`user_id`)** 입력 후 「임시 비밀번호 발급」
- 성공 시 **임시 비밀번호**를 화면에 1회 표시 (재조회 API 없음 — 안전한 경로로 사용자에게 전달)
- `MANAGER` 등 비관리자는 API·화면 링크로 접근 불가

## API

| 메서드·경로 | 용도 |
|-------------|------|
| `POST /api/v1/admin/users/{userId}/password-reset` | 본문 없음. 대상 `om_user_m.user_id`에 bcrypt 비밀번호 반영, `password_fail_cnt=0`, `user_status`가 `LOCKED`이면 `ACTIVE`로 복구 |
| `GET /api/v1/auth/me` | `data.gradeCd` 로 프론트에서 ADMIN 여부 판별 |

### 응답

- 성공: `data.userId`, `data.temporaryPassword` (평문 1회)
- 실패: `ERR_USER_NOT_FOUND` (대상 없음·삭제됨)
- 권한 없음: HTTP 403 (Spring Security, `ADMIN` 아님)

## 프로세스 흐름

1. 관리자가 로그인하여 세션에 `grade_cd=ADMIN` 이 반영된 상태인지 확인 (`/auth/me` 의 `gradeCd`).
2. 초기화 화면에서 대상 `user_id` 제출.
3. 서버는 `om_user_m` 행 존재·`is_deleted=false` 인지 확인 후 `user_info.password` 갱신, 실패 횟수 초기화, 잠금 해제(위 규칙).
4. 응답의 임시 비밀번호를 관리자가 사용자에게 전달 → 사용자는 로그인 후 **반드시 비밀번호 변경**하도록 안내(변경 UI는 별도 구현).

## DB

- `om_user_m.user_info.password` (bcrypt), `password_fail_cnt`, `user_status` — DDL `docs/ddl/om_user_m.sql` 참고.

## 주요 파일

- `backend/.../admin/AdminUserPasswordController.java` — `@PreAuthorize("hasRole('ADMIN')")`
- `backend/.../config/MethodSecurityConfig.java` — `@EnableMethodSecurity`
- `backend/.../mapper/OmUserAuthMapper.xml` — `adminResetPassword`
- `frontend/src/pages/AdminPasswordResetPage.tsx`
- `frontend/src/api/adminUsers.ts`
- `frontend/src/components/AppLayout.tsx` — ADMIN 일 때 링크

## QA 체크리스트

- [ ] `MANAGER` 로그인 시 해당 링크가 없고, API 직접 호출 시 403인가
- [ ] 존재하는 사용자에 대해 임시 비밀번호로 로그인되는가
- [ ] `LOCKED` 사용자 초기화 후 `ACTIVE` 및 실패 횟수 0인가
- [ ] 삭제·없는 아이디는 `ERR_USER_NOT_FOUND` 인가
- [ ] 로그인 화면에 분실 시 관리자 안내 문구가 보이는가
