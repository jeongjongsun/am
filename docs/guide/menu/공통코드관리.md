# 공통코드 관리

## 개요

`OM_CODE_M`에 등록된 **코드 그룹(대분류)**과 **그룹별 코드값(하위)**을 조회·등록·수정하는 화면입니다. 모든 업무 화면의 드롭다운·필터 등에서 참조하는 공통코드를 여기서 유지합니다.

- **삭제**: 행을 **물리 삭제하지 않습니다.** 사용 중지는 `code_info.use_yn = N` 으로 처리합니다.

## 경로·구성

| 경로 | 설명 |
|------|------|
| `/home/service/common-code` | 공통코드 관리 본화면 (단일 그리드 + 모달) |

> 이전 경로 `/home/admin/common-code` 는 동일 화면으로 리다이렉트됩니다.

## 화면 구성

1. **단일 그리드 ([AG Grid](https://www.ag-grid.com/react-data-grid/getting-started/) Community)**  
   - **대분류** 행(`main_cd = 'CODE'` 의 `sub_cd` = 그룹 ID)과 **하위 코드** 행(해당 `main_cd` 아래 `sub_cd`)을 **동일한 표**에 표시한다.  
   - 초기에는 **대분류 행만** 표시한다.  
   - 하단 **페이징**은 사용자 정보 화면과 동일하게 `DataGridPaginationFooter`를 사용하며, 페이지 크기 옵션은 `100, 1000, 5000, 10000`이다(클라이언트 슬라이스).

2. **열 구성 (요약)**  
   - **수정**: 첫 열에서 대분류·하위 각각 **모달 수정**.  
   - **관리**: 대분류 행에 **`+` / `−` 버튼**(펼침·접기)과 **하위 등록** 버튼.  
   - **구분**: 대분류/하위를 Phoenix `badge` 로 표시. **사용·순서** 등은 헤더·값 가운데 정렬.

3. **검색 영역**  
   - **공통코드**(그룹 ID·하위 `sub_cd`) 및 **코드명**(`code_nm` 다국어 값)에 대해 부분 일치 검색.  
   - 검색어가 있을 때는 모든 그룹에 대해 하위 목록을 조회한 뒤, 매칭되는 행만 표시한다(그룹·하위 혼합).

4. **등록·수정**  
   - **대분류 등록**: 상단 버튼 → **모달**.  
   - **그룹 수정** / **하위 수정**: 각 행의 편집 → **모달**.  
   - 표시 순서(`disp_seq`)는 모달에서 수정한다.

## API

| 메서드·경로 | 용도 |
|-------------|------|
| `GET /api/v1/admin/common-codes/groups` | 코드 그룹 목록 (`CODE` 하위) |
| `GET /api/v1/admin/common-codes/{mainCd}/items` | 해당 그룹의 코드값 목록 (`mainCd`는 `CODE` 불가) |
| `POST /api/v1/admin/common-codes/groups` | 그룹 등록 (`main_cd=CODE`, `sub_cd=groupId`) |
| `POST /api/v1/admin/common-codes/{mainCd}/items` | 코드값 등록 |
| `PUT /api/v1/admin/common-codes/{mainCd}/{subCd}` | 코드명·`use_yn`·`disp_seq` 수정 (그룹 메타·코드값 공통) |
| `PUT /api/v1/admin/common-codes/{mainCd}/items/order` | 본문 `{ "orderedSubCds": ["A","B",...] }` 로 표시 순서 일괄 저장 |

응답은 프로젝트 공통 `ApiResponse` (`success`, `data`, `message`, `code`). 실패 시 `code` 예: `ERR_COMMON_CODE_DUPLICATE`, `ERR_COMMON_CODE_GROUP_UNKNOWN` 등.

## DB·데이터

- 테이블: `om_code_m` ([om_code_m.sql](../../ddl/om_code_m.sql))
- 복합 PK: `(main_cd, sub_cd)`
- `code_nm`: JSONB 다국어 (`ko`, `en`, …)
- `code_info`: JSONB — `use_yn` (`Y`/`N`), `disp_seq` (정렬)

## 권한·보안

- REST URL은 `/api/v1/admin/common-codes/**` 이며, 백엔드는 `@PreAuthorize("hasRole('ADMIN')")` 등으로 보호한다. 비인가·만료 시에는 공통 인증 정책(401/403)을 따른다.

## 프로세스 흐름

1. 로그인 후 사이드 메뉴 **서비스 운영 정보 → 공통코드**로 진입.
2. 그리드에 대분류만 표시 → `+`로 하위를 펼친다.
3. 등록·수정은 모달에서 저장 → React Query 캐시 무효화 후 그리드 데이터 갱신.
